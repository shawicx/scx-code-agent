# Risks

## 成本风险

| 风险 | 严重度 | 原因 |
|------|--------|------|
| LLM 调用成本爆炸 | **高** | 请求数 = 文件数 × 3 专家，大项目可达数百次 |
| 并发 API 配额耗尽 | **高** | 最大 15 并发，可能超出 provider 限制 |

**场景**: 审查 100 个文件 → 300 次 LLM 调用。按 Claude Sonnet ~$3/M input tokens 计算，单次审查成本可能超过 $10。

## 无限循环

**不存在**。Graph 是 DAG，无条件边，无循环。

## Token 爆炸

**单次调用内可控**。每次 LLM 调用发送单个文件内容 + prompt。

| 风险场景 | 可能性 |
|----------|--------|
| 单文件过大超出 context | 中 |
| 累积 token 爆炸 | 低（无累积） |

## Prompt Injection

| 风险 | 严重度 | 原因 |
|------|--------|------|
| 被审查代码注入 LLM | **中** | 代码内容直接拼入 prompt |

**场景**: 被审查的代码文件包含类似 `]` 或恶意 JSON 结构，干扰 LLM 输出格式。

## Tool 越权

**不存在**。Agent 不具备执行代码、修改文件、访问网络的能力。纯分析输出。

## Hallucination

| 风险 | 严重度 | 原因 |
|------|--------|------|
| 虚构代码问题 | **高** | LLM 可能报告不存在的行号或问题 |
| 错误行号 | **高** | diff 模式下行号映射可能不准确 |

## JSON 解析失败

| 风险 | 严重度 | 原因 |
|------|--------|------|
| LLM 输出非 JSON | **高** | 尽管有修复逻辑，仍可能失败 |
| 修复逻辑引入错误 | **中** | `_repair_json()` 可能改变 JSON 语义 |

## Graph 死循环

**不存在**。DAG 拓扑，无条件边。

## Retry 风暴

| 风险 | 严重度 | 原因 |
|------|--------|------|
| LLM 持续返回格式错误 | **中** | tenacity 最多重试 3 次，但每个文件独立重试 |

**场景**: 如果模型持续返回 Markdown 代码块包裹的 JSON，3 次重试后放弃该文件（`review_code` 返回空列表）。

**注意**: `review_code()` 方法上虽然有 `@retry`，但内部 catch 了所有异常返回 `[]`。实际上重试只在外层触发，内层 `except Exception` 会吞掉错误。

## Context 膨胀

**不存在**。无累积 context，每次 LLM 调用独立。

## 最危险的节点

| 节点 | 危险原因 |
|------|----------|
| 3 个 expert | LLM 调用量大，成本不可控，输出质量依赖模型 |
| coordinator | diff 解析依赖正则，边界情况可能遗漏文件 |

## 最危险的工具

| 工具 | 危险原因 |
|------|----------|
| `LLMClient.review_code()` | 成本高、延迟高、输出不可靠 |

## 最容易失控的 Prompt

| Prompt | 失控原因 |
|--------|----------|
| `base.md` | LLM 可能不遵守 JSON 格式约束 |
| 3 个 role prompt | 可能过度报告或遗漏关键问题 |
